# SEO Optimization Design for CopiioAI GitHub Repository

## Goal

Drive functional/scenario keyword traffic to the CopiioAI Chrome extension via GitHub repository content. Target users who search for natural language data access tools, real-time web data solutions, and specific data source integrations — not brand-aware users.

## Positioning

CopiioAI is a **natural language interface for accessing real-time internet data**. Not an API testing tool. The core differentiator is the ability to query specific live data sources (Reddit, FRED, Seeking Alpha, TikTok, IMDb, Bloomberg, etc.) that general-purpose AI tools cannot access.

## Content Template per Document

Each doc follows this structure:
1. H1 with primary keyword
2. One-paragraph description reinforcing the keyword
3. Real example queries (from user's confirmed list), formatted as:
   ```
   > "show me latest hacker news"
   ✓ CopiioAI fetches live Hacker News top stories instantly
   ```
4. Brief "How CopiioAI handles this" section
5. CTA linking to Chrome Web Store and README

## Key Differentiators to Emphasize in Content

- **Real-time**: data is live, not cached or trained-in
- **Specific sources**: named data sources (FRED, Reddit, IMDb, etc.) not generic web search
- **No coding**: works for non-developers
- **Chrome extension**: works in the browser where users already are
